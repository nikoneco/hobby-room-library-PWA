# 検索結果シリーズ集約UI 仕様

作成日: 2026-07-10
状態: 実装・検証・本番反映済み

## 目的

通常検索および詳細検索で、同じシリーズの巻が続けて並ぶ状態を整理する。

利用者が次の両方を短時間で把握できるようにする。

- 検索条件に一致したシリーズの種類
- そのシリーズで検索に一致した所蔵巻

例として、`題材: 医療` の検索では『フラジャイル』全巻を1つのシリーズカードとして扱う。`フラジャイル 1` の検索では、部分一致した1巻と10巻から19巻を同じシリーズカードにまとめる。

## 適用範囲

| 画面・操作 | シリーズ集約 |
| --- | --- |
| 通常検索のタイル表示 | 適用する |
| 通常検索のリスト表示 | 適用する |
| 詳細検索のタイル表示 | 適用する |
| 詳細検索のリスト表示 | 適用する |
| ランダム表示 | 適用しない。巻単位のまま |
| 本棚表示 | 適用しない。巻単位のまま |
| 本詳細の「このシリーズを見る」 | 既存挙動を維持。所蔵全巻を表示 |

新しい表示形式の追加は行わない。タイルとリストはレイアウトの違いとして維持し、検索結果の単位だけを本からシリーズへ変更する。

## 集約ルール

1. 検索APIが返した検索結果だけを対象に、`seriesKeyAuto` でグループ化する。
2. 同じ有効な `seriesKeyAuto` を持つ検索結果が2冊以上ある場合だけ、1つのシリーズカードにまとめる。
3. 検索結果内で1冊しかない本は、既存どおり単独の本カードとして表示する。
4. `seriesKeyAuto` が空の本、または暫定キー（`__extra__` 系）の本は、題名が似ていても集約せず単独表示する。
5. 題名文字列だけを使った補完グループ化は行わない。別シリーズの誤結合を防ぐためである。

## 並び順と代表情報

- 検索結果の元の順番を基準にする。
- シリーズカードは、そのシリーズの検索一致本が最初に現れた位置に置く。
- シリーズカードの代表画像は、そのシリーズ内で検索結果に最初に現れた本の表紙を使う。
- シリーズカードを開いた後の巻カードも、検索結果の元の順番を維持する。

例: `フラジャイル 1` で検索した場合、シリーズ内の並びは `1巻, 10巻, 11巻, ... 19巻` とする。

## タイル・リストの表示内容

### シリーズカード

- 代表表紙
- シリーズ名（`seriesSearchTitle` を優先）
- `検索一致 N冊` の冊数表示

所蔵全巻数はここには混在させない。検索結果で一致した巻数を示すカードと、全巻を見る操作の意味を分ける。

### 単独の本カード

既存の本カード／リスト行の表示と操作を維持する。

### 検索結果件数

検索結果のステータスには、集約後のシリーズ数と元の検索一致冊数を併記する。

```text
8シリーズ・20冊
```

## シリーズカードを開いた後の導線

シリーズカードをタップすると、検索結果専用のシリーズ一覧ポップアップを開く。

```text
検索結果
  → シリーズカード
    → 検索一致した巻の横並びカード
      → 本の詳細
```

ポップアップの仕様は次のとおり。

- 見出しにシリーズ名と `検索一致 N冊` を表示する。
- 横スクロールで、検索に一致した巻だけを並べる。
- 各巻カードをタップすると既存の本詳細ポップアップを開く。
- 本詳細から戻ると、同じ検索条件で絞られたシリーズ一覧ポップアップへ戻る。
- キーボードのEscape、閉じるボタン、背景タップの挙動は既存モーダルと揃える。

## 既存の「このシリーズを見る」との区別

本詳細にある既存の「このシリーズを見る」は変更しない。

| 操作 | 表示する本 |
| --- | --- |
| 検索結果のシリーズカード | 今回の検索に一致した巻だけ |
| 本詳細の「このシリーズを見る」 | 所蔵しているシリーズ全巻 |

検索結果のシリーズカードから全巻取得APIを呼んではならない。検索条件を勝手に広げないためである。

## 実装方針

- 検索APIの結果配列をクライアント側でグループ化する。
- 既存の `seriesKeyAuto`、`seriesSearchTitle`、`isExtraSeries` を利用する。
- 検索結果専用のシリーズポップアップには、グループ化前の検索一致本配列をそのまま渡す。
- 本詳細の既存シリーズポップアップと検索結果専用ポップアップは、データ取得元と戻り先を分離する。
- 既存の `getBooksBySeriesKey` は本詳細からの全巻表示にのみ残す。
- 検索結果のDOM数を減らすため、先行実装済みの段階描画と両立させる。

## 受け入れ条件

1. 同シリーズの検索一致本が2冊以上あれば、タイルとリストではシリーズカード1件に集約される。
2. 1冊しか一致しない本は、本カードのまま詳細を直接開ける。
3. `フラジャイル 1` の検索で、1巻および10巻から19巻が同じシリーズカードに入る。
4. シリーズカードの表紙は、検索結果内で最初に出る巻の表紙である。
5. 検索結果専用ポップアップには、検索に一致した巻だけが横並びで出る。
6. 本詳細の「このシリーズを見る」は、検索条件に関係なく従来どおり所蔵全巻を表示する。
7. ランダム表示と本棚表示は、従来どおり巻単位で表示される。
8. シリーズキーなし・暫定キーの本は誤って集約されない。
9. 検索結果ステータスにシリーズ数と検索一致冊数の両方が出る。
10. iPhone、iPad、PCでタイル・リスト・ポップアップの操作が可能である。

## 手動テスト観点

- 同シリーズ全巻が一致する題材／ジャンル検索
- 1巻と10巻以降だけが一致する部分一致検索
- 同シリーズ内で1冊だけが一致する検索
- 複数シリーズと単巻本が混ざる検索
- `seriesKeyAuto` が空の本を含む検索
- 暫定シリーズキーを持つ本を含む検索
- タイルとリストを切り替えた後の集約状態
- 本棚表示、ランダム表示に集約が混入しないこと
- 検索結果シリーズ一覧 → 本詳細 → 戻る の遷移
- 本詳細 → このシリーズを見る の全巻表示が検索結果専用一覧と混ざらないこと
- 大量一致検索時のiPhone Safariでのスクロール、横スクロール、メモリ使用量

## 実装記録

2026-07-10に次の内容を実装した。

- 通常検索・詳細検索のタイル／リストを、検索結果内の `seriesKeyAuto` 単位で集約
- 代表画像に検索結果内の先頭一致巻を採用
- 1冊一致、シリーズキーなし、`__extra__` 系を単独表示
- 検索結果専用の横スクロール巻一覧ポップアップを追加
- 本詳細から検索一致巻一覧へ戻る導線を追加
- 既存の「このシリーズを見る」と `getBooksBySeriesKey` の全巻表示を維持
- 検索結果件数を `Nシリーズ・M冊` 形式に対応
- ランダム表示と本棚表示を巻単位のまま維持
- シリーズ集約、代表巻、並び順、暫定キー、検索専用ポップアップの回帰テストを追加
