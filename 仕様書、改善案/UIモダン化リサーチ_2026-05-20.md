# 趣味部屋図書館 UIモダン化リサーチ

作成日: 2026-05-20  
対象: `index.html`, `style.css.html`, `script.js.html` の見た目・体験改善案  
前提: この資料ではコード変更は行わない。実装前に方向性を固めるためのリサーチと提案に限定する。

## 1. 目的

趣味部屋図書館は、漫画・書籍を検索し、カード表示、リスト表示、本棚表示で眺められる個人用ライブラリである。現在のUIはダーク背景、シアン系グロー、カード、本棚モード、詳細モーダルなど、すでに「趣味の部屋」らしい世界観を持っている。

一方で、より洗練された現代風の印象にするなら、単に派手な装飾を増やすよりも、次の方向が相性がよい。

- 情報量の多い検索アプリとして、密度・視線誘導・状態表示を整理する。
- 本棚アプリとして、表紙画像の一覧性と「眺めて楽しい」体験を強める。
- 現在のシアン発光を抑え、上質な暗色ベースに少数のアクセントカラーを使う。
- コンポーネントの余白、角丸、影、文字サイズをルール化して、画面全体の統一感を出す。
- 詳細モーダルと本棚表示を、アプリの「気持ちいい中心体験」として磨く。

## 3. 現状UIの印象

コードを読んだ範囲では、現在の見た目は次の特徴がある。

- 背景は濃紺のラジアルグラデーション。
- ボタン、チップ、カードにシアン・ブルー系の発光表現が多い。
- 検索ヘッダーは大きなロゴを持ち、スクロール後に縮小する。
- 表示モードはタイル、リスト、本棚の3種類。
- 本棚表示には通常の棚表示と、より没入感のある暖色系の本棚モードがある。
- 詳細モーダルは表紙画像を大きく見せ、シリーズ移動や外部リンクも持つ。

良い点:

- 個人の趣味部屋らしい強い世界観がある。
- 本棚表示、棚マップ、シリーズ導線など、機能面に独自性がある。
- 表紙画像を中心にした体験は、漫画・書籍管理と相性がよい。

洗練を妨げやすい点:

- シアン発光、グラデーション、影、角丸の使い方がやや多く、視覚的な主張が散らばりやすい。
- ボタンやカードの色が強く、表紙画像よりUI装飾が目立つ場面がある。
- 検索フォーム、チップ、切替ボタン、CTA、本棚UIの見た目がそれぞれ別人格に見えやすい。
- ダークUIとしては、黒に近い面と明るいアクセントの差が強く、長時間見ると少し疲れる可能性がある。

## 4. 目指す方向性

### 推奨コンセプト

「静かな書庫に、検索の速さと表紙を眺める楽しさがあるUI」

雰囲気は、サイバー風の強い発光から少し離し、現代的なダークライブラリに寄せる。完全な業務ツールにはせず、表紙、本棚、棚マップの楽しさを残す。

キーワード:

- 静か
- 高密度だが窮屈ではない
- 表紙が主役
- 検索は速く、条件は見失わない
- 本棚は少し没入感がある
- 装飾より質感

## 5. ビジュアル方針

### カラーパレット

現在の「濃紺 + シアン発光」から、次のような構成に寄せる。

| 役割 | 方向性 | 例 |
| --- | --- | --- |
| 背景 | 黒ではなく、青みを少し含むチャコール | `#11151c`, `#171b24` |
| 面 | 背景より少し明るいスレート | `#1d2430`, `#252d3a` |
| 境界線 | 低コントラストの薄い線 | `rgba(255,255,255,0.08)` |
| 主アクセント | 落ち着いたシアンまたはブルー | `#4fb6d8`, `#5aa8ff` |
| 副アクセント | 本棚らしい琥珀色 | `#d6a45f`, `#f0c37a` |
| 成功・選択 | 彩度を抑えたミント | `#69c6a3` |
| 警告・未整理 | 控えめなアンバー | `#e0a84f` |

ポイント:

- シアンは「検索」「選択中」「フォーカス」など重要な状態だけに使う。
- 本棚表示では琥珀色を少し使い、表紙と木棚の雰囲気を出す。
- 背景に大きなグラデーションを置く場合も、彩度を低くして表紙画像を邪魔しない。

### タイポグラフィ

日本語UIでは、文字の読みやすさが洗練感に直結する。

- ベースフォントは `Noto Sans JP`, `system-ui`, `Segoe UI`, `sans-serif` などに寄せる。
- タイトルはやや太め、本文とメタ情報は軽めにして階層を作る。
- 文字サイズは画面全体でルール化する。
  - 画面タイトル: 24-32px
  - セクション見出し: 18-22px
  - カードタイトル: 14-16px
  - メタ情報: 12-13px
  - チップ: 11-12px
- 文字の発光や強いテキストシャドウは最小限にする。

### 余白・角丸・影

余白と角丸を統一すると、急に「作り込まれたアプリ」に見える。

- 余白単位は 4px または 8px 基準にする。
- カード角丸は 8px から 10px 程度に抑える。
- モーダルや大きなパネルだけ 16px から 20px を使う。
- 影は発光よりも、薄い境界線 + 柔らかい落影にする。
- ホバー時の変化は、強い光ではなく、少し浮く・境界線が明るくなる程度にする。

## 6. 画面別の改善案

### 6.1 トップ・検索エリア

現在はロゴが大きく、検索フォームが中心にある。ここはアプリの第一印象なので、次のようにすると洗練される。

- ロゴ下の検索フォームを、横長の検索バーとして主役化する。
- 通常検索、ランダム検索、詳細検索を同じ重さで並べず、主操作を「検索」、副操作をアイコンまたは控えめなボタンにする。
- 詳細検索は展開時にカード風ではなく、薄い区切り線のあるパネルとして扱う。
- 検索条件チップは左上固定より、検索バー直下または結果上部に置くと状態を追いやすい。
- 検索前の画面には「本棚を見る」を大きく置くより、検索バーと本棚ショートカットを一体化した導線にする。

推奨レイアウト:

1. 上部に小さめロゴとアプリ名。
2. 中央に大きな検索バー。
3. 検索バー右に検索ボタン。
4. 下に「詳細」「ランダム」「本棚」の小さなアクション。
5. 条件チップと対象件数は検索バー下に横並び。

### 6.2 検索結果カード

カード表示では表紙が主役になるようにする。

- カード背景のグラデーションを弱め、表紙画像の色を引き立てる。
- タイトル、作者、棚位置、発売日を情報階層で分ける。
- 価格や棚位置などのメタ情報は、全部同じ文字列で並べず、小さなラベル風にする。
- ジャンルチップは色数を抑え、主要ジャンルだけ見せる。
- ホバー時は発光ではなく、カードが 2px 程度浮く、境界線が明るくなる、表紙が少し拡大する程度にする。

カード構成:

- 表紙画像
- タイトル 2行まで
- 作者
- メタ行: 棚 / 発売日 / 状態
- ジャンルチップ

### 6.3 リスト表示

リスト表示は「大量の蔵書を素早く確認する」ためのモードにする。

- 行の高さを安定させ、タイトル・作者・棚位置を同じ列に揃える。
- 表紙サムネイルは小さくても判別できるサイズを維持する。
- 詳細情報はクリック展開より、右側に重要情報を固定表示するほうが一覧性が高い。
- スマホでは2段構成にして、表紙 + タイトル + メタ情報を読みやすくする。

### 6.4 本棚表示

本棚表示はこのアプリの独自価値なので、最も「カッコよく」する余地がある。

- 暖色系の没入モードは残しつつ、木目や影の主張を少し抑える。
- 棚グループの見出しはチップ風ではなく、書庫の案内板のように扱う。
- 表紙の高さを揃え、横スクロール時のリズムを作る。
- 空の棚や配置未定は、控えめなプレースホルダーで見せる。
- 棚マップはモーダルだけでなく、現在位置を示すナビゲーションとしても使える。

本棚モードの方向:

- 背景: チャコール + ごく薄い木棚テクスチャ感
- 棚: 暗い木材風の面、ただし彩度低め
- 表紙: 主役。影は自然に
- ナビ: 上部に固定、琥珀色アクセント

### 6.5 詳細モーダル

詳細モーダルは「この本を開いた」感覚がある場所なので、かなり重要。

- 表紙を左、情報を右に分ける2カラムにする。スマホでは縦積み。
- タイトル、作者、巻数、シリーズ導線を上部にまとめる。
- あらすじは折りたたみ式のままでもよいが、本文幅と行間を読みやすくする。
- 外部リンクは同じ色の丸ボタンではなく、用途別の控えめなボタンにする。
- 前後移動は矢印だけにして、ボタンの存在感を抑える。

### 6.6 詳細検索

詳細検索は「高機能だけど怖くない」ことが大切。

- 入力欄をすべて同じ重さで並べず、よく使う項目と補助項目を分ける。
- タイトル、作者、出版社、ジャンル、状態、発売期間の順に整理する。
- 条件クリアは危険操作ではないが、検索ボタンより目立たせない。
- 選択済み条件はチップで常に見えるようにする。

## 7. 具体的なデザイン案

### 案A: Modern Dark Library

最もおすすめ。現在の雰囲気を活かしながら、発光を抑えて上質にする。

- 背景は青黒いチャコール。
- アクセントはシアン 1色 + 本棚の琥珀色。
- カードは薄い境界線、低い影、表紙重視。
- 検索フォームはガラス感を少しだけ使う。
- 本棚モードは暖色を残して差別化する。

向いている理由:

- 現在のコード・世界観からの距離が近い。
- 漫画・書籍の表紙が映える。
- 個人用ライブラリとして落ち着きと楽しさの両方が出る。

### 案B: Editorial Book Archive

雑誌・書店・レビューサイト寄りの雰囲気。

- 明るめの背景、黒文字、紙のような余白。
- 表紙とタイポグラフィを大きく扱う。
- ランキングやおすすめ棚のような見せ方が得意。

懸念:

- 現在のダークな趣味部屋感から大きく変わる。
- Google Apps Scriptの既存画面に入れると、実装範囲が広くなる可能性がある。

### 案C: Compact Collection Tool

業務システムに近い、密度重視のUI。

- 検索、絞り込み、一覧、ソートを最優先。
- 色はかなり控えめ。
- リスト表示とテーブル風の一覧が主役。

懸念:

- 便利だが、趣味部屋らしい楽しさは弱くなる。
- 「カッコイイ」より「管理しやすい」に寄りやすい。

結論: まずは案Aを軸にするのがよい。案Aをベースに、リスト表示だけ案Cの密度を取り入れる。

## 8. 実装するなら優先したい順番

### Phase 1: 見た目の土台を整える

- カラートークンを定義する。
- フォント、文字サイズ、行間を整理する。
- 角丸、境界線、影、ホバー表現を統一する。
- ボタンの種類を primary / secondary / ghost / danger などに分ける。

効果:

- 画面全体の統一感が最短で上がる。
- 後続の改修が楽になる。

### Phase 2: 検索体験を磨く

- 検索バーを主役にする。
- 詳細検索パネルの密度と余白を調整する。
- 条件チップと対象件数を結果エリアに近づける。
- 表示切替ボタンを現代的な segmented control にする。

効果:

- 初見でも迷いにくい。
- アプリとしての完成度が上がる。

### Phase 3: カード・リスト・モーダルを磨く

- 表紙画像を主役にしたカードにする。
- リスト表示は列揃えと展開挙動を整理する。
- 詳細モーダルを2カラム化し、情報階層を作る。
- 外部リンクボタンを控えめに統一する。

効果:

- 検索後の気持ちよさが上がる。
- 蔵書量が増えても見やすい。

### Phase 4: 本棚モードを特別な体験にする

- 本棚モードの暖色テーマを整理する。
- 棚グループ、段、表紙のリズムを整える。
- 棚マップをより自然なナビゲーションにする。
- スクロール・ホバーの動きを控えめに整える。

効果:

- 趣味部屋図書館ならではの魅力が強くなる。
- ただの検索アプリではなく「自分の書庫」になる。

## 9. UI部品ごとのルール案

| 部品 | ルール |
| --- | --- |
| ボタン | 主ボタンは1画面1種類。副操作は輪郭または透明ボタンにする。 |
| チップ | 条件表示・ジャンル表示・状態表示で色を変えすぎない。意味のある色だけ使う。 |
| カード | 背景は静かに。表紙、タイトル、作者を優先する。 |
| モーダル | 情報量が多いので、背景・表紙・本文・アクションを明確に分ける。 |
| フォーム | ラベルは常に表示。プレースホルダーだけに頼らない。 |
| 表示切替 | タイル・リスト・本棚は segmented control にする。 |
| ローディング | 大きなスピナーより、検索バーや結果エリアに自然な進行表示を置く。 |
| 空状態 | 「該当なし」を冷たく出すだけでなく、条件を外す導線を出す。 |

## 10. 実装前チェックリスト

- 主要画面を3つに絞ってラフを作る: 検索前、検索結果、詳細モーダル。
- 主要表示モードを3つ確認する: カード、リスト、本棚。
- 色を7種類以内に整理する。
- 文字サイズを6段階以内に整理する。
- 影と発光を使う場所を限定する。
- スマホ幅で検索フォームとボタンが詰まらないか確認する。
- 本棚モードで表紙が主役になっているか確認する。
- 詳細モーダルであらすじが読みやすいか確認する。
- コントラスト比を確認する。

## 11. まとめ

趣味部屋図書館を洗練された現代風にするには、「強い発光のダークUI」から「静かで質感のあるダークライブラリUI」へ寄せるのがよい。

最初にやるべきことは、派手な新機能追加ではなく、色・余白・文字・影・ボタンのルール化である。その上で、検索バー、カード、詳細モーダル、本棚モードを順に磨くと、今の独自性を残したまま、かなりカッコよくできる。

おすすめの最終イメージ:

> 表紙が映える暗い書庫。検索は速く、条件は見失わず、本棚を眺める時間が少し楽しい。UIは静かだが、細部に気が利いている。
